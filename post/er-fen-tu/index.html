二分图

2021-05-06 6 min read

一、定义

二分图（Bipartite graph），又称二部图。简单来说，即图中的节点有两个集合组成，且两个集合内部没有连边的图。

我们称这两个点集一个叫左部，一个叫右部。左部中的点叫左部点，右部中的点叫右部点。

二、性质

  • 二分图的每一条边一定连接两个不同集合中的点。

  • 二分图中没有节点个数为奇数的环。

    因为每条边都是连接不同集合的点，因此经过一条边一定是从一个集合走到另一个集合，要回到原来的集合形成环一定经历偶数条边。

三、判定

（1）方法

根据上面的性质，我们可以遍历整张图，若发现奇环，该图就不是二分图，否则的话就是。

（2）算法

我们可以用染色法。流程大致如下：

  1. 用黑白两色标记图中的节点，当一个节点被标记后，它的所有相邻节点都应该被标记为其相反颜色；

  2. 若标记过程中出现冲突，则说明图中有奇环。

二分图染色一般用 dfs 做，时间复杂度为 O(n+m)O(n+m)O(n+m)。

（3）代码

void dfs(int u, int c) {
    if(!flag)
        return;
	col[u] = c;
	for(int i = he[u]; i; i = e[i].nxt) {
		int v = e[i].to;
		if(col[v] == c) {
			flag = false;
			return;
		}	
		if(!col[v])
			dfs(v, 3 - c);
	}	
}

四、应用

（1）二分图最大匹配

1. 定义

  • 匹配

    图 GGG 的一个匹配是 GGG 的一个边集 E0E_0E0​，满足 E0E_0E0​ 中的所有边没有 公共顶点。

    下图中的红线部分即为图的一个匹配。

  • 最大匹配

    一个图的所有匹配中，匹配数最多的匹配。

  • 完美匹配

    所有点都属于匹配，同时符合最大匹配。

2. 算法

  • 匈牙利算法

    匈牙利算法的过程是枚举每一个每一个左部点，然后枚举该左部点的出边，对于一个出点 vvv，若其没有被先前的左部点匹配，那么直接将 uuu 匹配 vvv，否则的话尝试让 vvv 原来匹配的左部点去匹配其他右部点，若匹配到了其他点，将 uuu 匹配 vvv，否则的话 uuu 匹配。

    算法时间复杂度为 O(n×e+m)O(n\times e + m)O(n×e+m)，nnn 是左部点个数，eee 是边数，mmm 是右部点个数。

3. 代码

bool dfs(int u, int tag) {//二分图匹配 
	if(book[u] == tag)
		return false;
	book[u] = tag;
	for(int i = he[u]; i; i = e[i].nxt) {
		int v = e[i].to;
		if(!match[v] || dfs(match[v], tag)) {
			match[v] = u;
			return true;
		}
	}
	return false;
}

4. 例题

P3386 【模板】二分图最大匹配

模板题。用匈牙利算法来做，枚举每一个左部点，若可以直接和右部点匹配的话就匹配，否则的话就让右部点原先的匹配对象去匹配另外的右部点，匹配不了的话没办法，当前枚举到的左部点就只能失配了。

至于如何判断失配，加入时间戳以提高效率。具体来说，在匹配当前左部点的时候时间戳就是当前左部点编号，搜索中给搜到的每一个左部点打上时间戳，若在搜索中搜到了一个点的标记等于时间戳，说明无法匹配了，也就失配了。

P1640 [SCOI2010]连续攻击游戏

关键在于建图。我们建立每个物品两个属性值连向其编号的单向边。因为属性值一定是连续的，所以我们从小到大枚举属性值依次判断是否可以匹配，若枚举到 iii 时无法匹配，则最多只能攻击 i−1i-1i−1 次。

CF1139E Maximize Mex

建立每个学生能力值连向其社团编号的单向边。求 mex⁡\operatorname{mex}mex 的最大值只需从小到大枚举能力值判断是否可以匹配，当我们枚举到 iii 时，若其无法匹配，则 mex⁡\operatorname{mex}mex 的最大值即为 iii。

每天都有学生退出社团，我们很难维护这种操作，正难则反，我们倒叙枚举删去的学生，统计完当天的答案后，就将当前枚举的学生加入图中。注意匹配和标记数组初始化应为 −1-1−1，因为 000 也是能力值。

P1129 [ZJOI2007] 矩阵游戏

发现图论题关键在于如何建图。本题是将每个 111 的横纵坐标相连，再跑二分图的匹配。为什么呢？我们会发现，交换行列其实不会改变什么，同行的还是同行，同列的还是同列，而我们要对角线都为 111，也就是 nnn 个不同行不同列的 111，那么开始若能完美匹配就满足条件了。

P1407 [国家集训队]稳定婚姻

很显然的一道二分图匹配，关键在于删边。

P2756 飞行员配对方案问题

裸的二分图最大匹配，不过注意数组一定要开够，还有直接记录匹配的点就行了。

P2055 [ZJOI2009]假期的宿舍

根据题意，我们可以将学生当作左部点，床当作右部点，然后就可以跑二分图的最大匹配了，其实还是板子，但是要根据题目要求来建边。
