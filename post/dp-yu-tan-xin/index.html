P3129 [USACO15DEC]High Card Low Card P

本题贪心的想法很容易想到：设 fif_ifi​ 表示从 111 到 iii 都用规则一赢的最多场数，gig_igi​ 表示从 iii 到 nnn 都用规则二赢的最多场数，那么答案就是 max⁡(fi+gi+1)\max(f_i+g_{i+1})max(fi​+gi+1​)。但是可能会出现一个数 xxx 在两个地方都被用到的情况，那为什么还是对的呢？因为这样一定有个 yyy 在两个地方都没出现，那么这个 yyy 一定可以取代一个地方的 xxx。

CF505E Mr. Kitayuta vs. Bamboos

最大值的最小值肯定用二分求解，那么问题就变成了怎么判定 mmm 天后 hhh 的最大值是否小于等于 midmidmid。这里用到很重要的技巧：倒着解决问题，因为 hhh 减少到 000 以下很难处理，因此我们把问题转化为初始时每棵树高度都为 midmidmid，每天可以选择一棵树加上 ppp，所有树都会砍掉 aia_iai​，需要保证竹子减少 aia_iai​ 后不会 <0<0<0，每次选最快小于 000 的拔高就行了。

CF727F Polycarp's problems

我们发现尽量用正数去填绝对值较小的负数更优，因此我们用堆维护负数贪心地做。然后我们处理出没消完的负数，这些就是留给 a0a_0a0​ 来消的，然后对于每一个询问都二分查找一下就行。

CF746F Music in Car

首先因为是连续的字串所以显然可以用双指针来做，那问题就是加入和删除元素时要怎么操作。我们建立两个 set，第一个存打折的另一个存不打折的，加入一个新元素 ara_rar​ 时我们先默认它是打折的，把它加入第一个 set 里，若 set 的大小大于 mmm 的话我们弹出 set 中最小的元素加入第二个 set 里。删除一个元素 ala_lal​ 的话，先判断它在哪个 set 里，这个可以通过比较和第一个 set 的首元素的大小得出，然后再操作。记住删去第一个 set 中元素后要把第二个 set 里最大的补到第一个里。

AT2335 [ARC069C] Frequency

我们发现先以数值大小为第一关键字，下标大小为第二关键字排序之后，iii 会被选到仅当前面所有数小于等于 aia_iai​，因此排序后我们扫一遍再统计答案就行了。

P3959 [NOIP2017 提高组] 宝藏

首先明确题意：要求构造一棵生成树，使得构造的代价最小，开发一条边的代价为这条边的边权 ×\times× 生成树的深度。看到 n≤12n\leq 12n≤12 我们可以想到状压 dp，我们设 fS,if_{S,i}fS,i​ 表示选择的点集为 SSS，生成树深度为 iii 的最小代价。接下来考虑怎么转移， 我们用 SSS 的子集 sss 来更新 SSS 的答案，具体来说我们预处理出所有状态能扩展出去的点，然后在用 sss 更新 SSS 的时候，我们枚举 sss 能扩展出去的点，在这些中找到边权最小的来更新。

CF605E Intergalaxy Trips

期望 dp。发现正着考虑会不知道到当前位置要走几步，因此我们倒着考虑，设 fif_ifi​ 表示从 iii 走到 nnn 的最小期望天数。转移的时候，对于一个 jjj，我们会走它当且仅当其他比它小的都没有出现，然后就可以用类似 dijkstra 的方法 O(n2)O(n^2)O(n2) 做了。
